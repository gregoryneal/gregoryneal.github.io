Overview

This website was built to be my personal website, mainly as a place to store my portfolio and projects. I have outlined my plan for deployment below. I wanted to work in a .Net environment and then deploy to a pure html/css/js static site. For that I had to split my build process in two:

  • local environment: my local development environment is built on top of the ASP .Net Core framework.

  • deployment environment: my deployment process consists of recursively crawling my locally hosted development site with wget, saving the output html, css, js and any images. Then it's as simple as deploying on github pages through my github account.

Difficulties

One of the things that I knew that I needed before even picked the framework was that I needed to be able to easily add posts and projects without doing a lot of extra coding. Ideally I wanted to be able to add <project-name>.html to my "projects" folder, it would automatically create a page at "site.com/projects/<project-name>". I found a solution through a custom controller, and the anchor tag helpers that let me explicitly specify asp-route-<data>.

Future

Backend wise I want to abstract some of my auto routing code to make it more modular. Then possibly turn it into a nuget package.

For the frontend I'm going to work on some of the design. I was aiming for a visual studio dark theme feel, but I think I can make it better.
