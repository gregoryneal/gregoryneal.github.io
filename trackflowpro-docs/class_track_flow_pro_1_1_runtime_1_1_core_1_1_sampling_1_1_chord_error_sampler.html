Map a single "quality" slider to the adaptive chord-error sampling parameters.

  • quality01: 0 = fastest/coarsest, 1 = slowest/finest.
  • scaleHint: recommended = road width (meters) or pipe diameter (meters). If <= 0, we fall back to 1.
  • curveLength: optional (for sample count estimates and clamping maxStep).

Tunables below are in relative-to-scale units by default, so the mapping is consistent across sizes

void TrackFlowPro.Runtime.Core.Sampling.ChordErrorSampler.RebuildPolylineArcLengths ( ref ChordErrorSampler.Polyline poly)
static

Ensure the Polyline arc length array is global.

Normally in ArcFrame you would take all of your IArcLengthCurve and stuff them into a CompositeCurve() then we could sample that instead for inherent global arc length array. But... I decided to not do that for some reason here. So I am using this method to ensure Polyline.S looks like [0, ..., L] instead of [0, ..., L0, 0, ..., L1, 0, ..., L2, ...]
